### `phase1_depthmap` — new
- ANSI color depth map for a single sensor
- 9-step color palette (white → red → orange → yellow → green → cyan → blue → dark)
- Block characters scale with distance (██ ▓▓ ▒▒ ░░ · · ·)
- Configurable 4×4 or 8×8 mode

### `phase2_dual_sensor` — updated
- Same SparkFun library port as phase1
- LPN pin sequence to assign unique I2C addresses at boot:
  1. Pull both LPN LOW (both sensors off bus)
  2. Wake Sensor A, call `begin(0x29)`, then `setAddress(0x2A)`
  3. Wake Sensor B, call `begin(0x29)` (it now has the bus to itself)

### `phase2_depthmap` — new
- Side-by-side ANSI depth maps for both sensors simultaneously
- **INT-pin-driven readiness** (`digitalRead(INT_x_PIN) == LOW`) as primary path;
  `isDataReady()` as I2C fallback only when INT is not LOW
- Fixed-width `snprintf` rendering to eliminate display corruption
- ANSI `\033[K` (erase-to-EOL) and `\033[J` (erase-to-bottom) for flicker-free updates
- **Sensor B watchdog:** if Sensor B is stale >3 s, its LPN pin is pulled LOW to
  isolate it from the bus; automatic re-initialisation attempted every 30 s
- Periodic full-screen clear every 5 s (guarantees recovery for late-connecting terminals)
- Freshness indicator per sensor: `[live]` / `[Nms ago]` / `[STALE]`

**Observed behavior (expected, not a bug):** Sensor B's bottom 2 rows consistently
read ~930–943 mm regardless of rider movement. These zones are seeing the
floor/stand through the 45° mount angle. The `MAX_VALID_MM = 800` filter in
Phase 3 excludes them from the position algorithm.

### `phase3_position` — new ⬅ Week highlight
Position mapping algorithm that fuses both sensors into a single 2D rider
position estimate.

**Algorithm:**
```
depth (F/B) = (dist_A + dist_B) / 2  ×  cos(45°)
lean  (L/R) = (dist_A − dist_B)      ×  cos(45°)
```

- `dist_A`, `dist_B` = inverse-distance-weighted average of all valid zones
  (zones outside `MIN_VALID_MM`–`MAX_VALID_MM` are excluded)
- Sign convention: **lean < 0 → leaning left** (toward A); **lean > 0 → leaning right** (toward B)
- Optional secondary lean signal from zone column centroid (`CENTROID_WEIGHT`, default 0)
- Joystick mapping: Joy X (lean) 0–255; Joy Y (depth) 0–255; both `constrain()`-clamped

**Terminal display:**
- Both raw zone grids (compact, same color palette as phase2_depthmap)
- Live 2D position box — `◆` marker moves with rider position:

```
  F ┌────────────────────────┐
    │                        │   A (left)   slant  412mm  9 zones
    │                        │   B (right)  slant  398mm  7 zones
    │           ◆            │
    │                        │   depth  286mm   lean  +10mm
    │                        │   Joy X (lean) :  133 / 255
    │                        │   Joy Y (depth): 156 / 255
  B └────────────────────────┘
    L                        R
```

## Architecture Decision — Dual-Core Integration Plan

After reviewing `magduino.ino` (USB HID gamepad) and the KiCad schematic/PCB:

- **Final target:** single unified `main.ino` on one Pico
- **Core 0:** HID gamepad loop (TMAG5170 Hall effect, ADS1115 throttle/brake,
  buttons, USB HID report) — target <30 ms latency
- **Core 1:** VL53L5CX polling loop + Serial JSON output to Mission Control
- **I2C assignment:** VL53L5CX sensors must share I2C0 (GP12/GP13) with ADS1115
  ADCs, bridged through the onboard TCA9406DC 3.3V↔1.8V level translator (U4)
- **GPIO for LPN/INT:** GP0, GP1, GP11, GP14–GP17, GP28 are unconnected on the
  PCB traces and accessible as castellated pads — available for ToF control signals
- **Current test-bench pins (GP2–GP7) conflict** with button assignments on the
  production PCB; will be remapped before integration

## Sensor Geometry Insight

The V-geometry mount was clarified:

```
    [A left, 45° inward]          [B right, 45° inward]
              ↘ 424mm slant  424mm ↙
                   → [Rider] ←
```

- **Using both sensors together** is required for reliable L/R lean detection.
  A single sensor at 45° cannot distinguish lean from depth change.
- `dist_A − dist_B` provides the lateral lean signal; `(dist_A + dist_B) / 2`
  provides the depth signal. Both are independent and orthogonal.
- Background floor zones (~930 mm) are filtered by `MAX_VALID_MM = 800`.
