## What Was Accomplished Today

### Root cause of the week-long instability — identified and resolved

**Cause:** Both Alibaba breakout boards carry an 8-pin resistor array (472 = 4.7kΩ)
providing pull-ups for SDA, SCL, INT, and LPN. With two boards on the same I2C bus
the combined pull-up resistance drops to ~2.35kΩ — below the reliable threshold for
400kHz I2C operation. Signal integrity degrades, causing intermittent NACK, dropped
frames, and bus lockups.

**Fix:** Dropped I2C clock from 400kHz to 100kHz in `phase3_raw`. At 2.35kΩ combined,
100kHz is well within spec. No hardware modification required. 4×4 @ 60Hz fits
comfortably within 100kHz I2C bandwidth (~6KB/s vs ~10KB/s available).

**Note for production:** The custom PCB uses a TCA9406DC level translator with its
own properly-sized pull-ups. The pull-up crowding issue does not exist on the
production board — 400kHz can be restored there.

### Bugs fixed

| Bug | Fix |
|-----|-----|
| `Wire.setTimeout(200)` around `begin()` was aborting VL53L5CX firmware upload (~2–4s) | Removed `Wire.setTimeout` calls entirely from `test_sensor_a`, `test_sensor_b`, `phase3_raw` |
| `Serial.flush()` in `redraw()` blocking loop when terminal not reading | Removed flush; replaced with `availableForWrite()` guard on all draw calls |
| Watchdog firing on first loop pass (B never got a frame) | Set `lastReadA/B_ms = millis()` after `startRanging()` in `setup()` |
| Watchdog pulling LPN_B LOW mid-transaction → SDA stuck LOW → A freezes | Added `recoverWire()` (Wire.end + Wire.begin) after every LPN isolation |
| Serial Plotter toggle hang | Guarded plotter `Serial.print` with `Serial.availableForWrite() > 64` |
| Auto-scaling in Serial Plotter made lean unreadable | Added `MIN:0 MAX:255` anchors; all channels mapped to 0–255 |

### New sketches

**`firmware/test_sensor_a/test_sensor_a.ino`**
Isolates Sensor A completely — LPN_B held LOW the entire run. Boots A at 0x29,
reassigns to 0x2A, streams plain-text distance readings to Arduino Serial Monitor.

**`firmware/test_sensor_b/test_sensor_b.ino`**
Isolates Sensor B completely — LPN_A held LOW the entire run. B stays at factory
address 0x29. Prints a diagnostic checklist if `begin()` fails. Used to confirm B
is healthy independent of A.

**`firmware/phase3_raw/phase3_raw.ino`**
Plain-text dual-sensor validator. No ANSI codes — works in Arduino Serial Monitor
and Serial Plotter. Outputs one human-readable line + one plotter line per frame:

```
#226693 | A  323mm  8z  B  320mm 11z |  227mm     +3mm | 129 193
MIN:0 MAX:255 Depth:193 Lean:129 DistA:95 DistB:91
```

Column reference:

| Column | Meaning |
|--------|---------|
| `#N` | Frame counter (Sensor A frames) |
| `A Xmm Yz` | Sensor A weighted average distance / valid zone count |
| `B Xmm Yz` | Sensor B weighted average distance / valid zone count |
| `Xmm` (5th) | Computed perpendicular depth `(A+B)/2 × cos45°` |
| `±Xmm` | Computed lateral lean `(A−B) × cos45°` |
| Last two ints | Joy X (lean, 0–255) and Joy Y (depth, 0–255) |
| Flag `!` on A/B | That sensor's data is stale (>500ms) |
| Flag `~` on B | B data not updated this frame but recently fresh |
| Flag `?` on B | B data old (watchdog may be active) |

### Hardware finding — board anatomy confirmed

The Alibaba VL53L5CX breakout boards contain:
- **8-pin resistor array (472 / 4.7kΩ)** — pulls up SDA, SCL, INT, LPN simultaneously
- **3-pin SOT-23 (662K)** — voltage regulator or transistor (do not remove)
- **4× 0402 decoupling capacitors**
- **3× individual 472 resistors** — LPN, INT, and one additional signal

Pull-up array confirmed via continuity probe. Trace cutting not feasible on this
board. 100kHz software fix preferred over hardware modification.

### Production board assessment

The `pico_breakout` PCB has no dedicated VL53L5CX connector. The TCA9406DC on the
board bridges Pico I2C0 (GP12/GP13) to the ADS1115 ADC bus. VL53L5CX sensors
connect via flying wires to J1/J2/J3 breakout header pads. LPN and INT signals
require 4 of the 8 free castellated pads (GP14–GP17 recommended).
